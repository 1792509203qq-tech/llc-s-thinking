### 二、掌握两大核心交互模式

#### 2.1 Composer 模式

**快捷键**：
- Windows/Linux: `Ctrl+K`
- Mac: `Cmd+K`

**功能特点**：
- 用于生成和修改代码
- 通过自然语言描述，Cursor 将根据描述生成或修改代码
- 适合批量代码生成和重构

**学习任务**：
- [ ] 使用 `Ctrl+K` 打开 Composer
- [ ] 尝试用自然语言描述生成一段代码
- [ ] 理解 Composer 的代码修改能力

#### 2.2 Chat 模式

**快捷键**：
- Windows/Linux: `Ctrl+Shift+L`
- Mac: `Cmd+Shift+L`

**功能特点**：
- 用于与 AI 助手交流
- 获取编码帮助和解释
- 适合问答、调试、代码理解

**学习任务**：
- [ ] 使用 `Ctrl+Shift+L` 打开 Chat 面板
- [ ] 尝试询问代码相关问题
- [ ] 理解 Chat 与 Composer 的区别和使用场景

### 三、熟悉上下文管理模式

**核心概念**：使用 `@` 符号引用代码、文件、文档等上下文内容，提供更准确的建议和分析

#### 3.1 @Files - 引用特定文件

**作用**：
- 引用项目中的特定文件
- 让 AI 理解特定文件的上下文
- 针对特定文件进行代码生成或修改

**使用场景**：
- 需要修改某个具体文件时
- 需要理解某个文件的逻辑时
- 需要基于某个文件生成相关代码时

**实践任务**：
- [ ] 在 Chat 中使用 `@Files` 引用一个文件
- [ ] 尝试基于引用的文件进行提问或修改

#### 3.2 @Folders - 引用整个文件夹

**作用**：
- 引用整个文件夹，提供更广泛的上下文
- 适合需要理解整个模块或功能时使用

**使用场景**：
- 需要理解某个模块的整体结构
- 需要跨文件进行代码重构
- 需要分析某个目录下的所有文件

**实践任务**：
